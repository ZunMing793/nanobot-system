# Skill 创建器详细指南

本文档包含创建和重构 skills 的完整规范和流程。

---

## 1. 设计原则

### 1.1 问题背景

| 问题 | 原因 | 后果 |
|------|------|------|
| bot 不读取 skill 内容 | 没有设置 `always: true` | bot "幻觉"输出，假装调用了 skill |
| SKILL.md 内容太长 | 所有内容放一个文件 | 全部 always:true 会占用太多上下文 |

### 1.2 解决方案

**精简 SKILL.md（只做钩子） + 详细内容拆分到 GUIDE.md + always: true**

| 文件 | 职责 | 加载方式 |
|------|------|----------|
| **SKILL.md** | 钩子：触发场景 + 执行前必读 + 详细指南索引 | `always: true` |
| **GUIDE.md** | 全部详细内容：格式、方法、示例、参考表格等 | 按需读取 |

**核心原则**：SKILL.md 不给任何可用的格式/步骤，逼 bot 必须读取 GUIDE.md

---

## 2. SKILL.md 标准结构（40-60行）

```yaml
---
name: skill-name
description: "触发场景描述（要详细，用于匹配用户输入）"
always: true
---

# Skill 名称

一句话说明这个 skill 做什么。

## 触发场景

- 场景1
- 场景2
- 场景3

---

## ⚠️ 执行前必读

**决定使用此 skill 后，必须先读取**：
`C:/Users/79345/.claude/skills/skill-name/GUIDE.md`

**禁止事项**：
- ❌ 不准偷懒不读取 skill 的其他详细文件
- ❌ 不准直接用 SKILL.md 中的精简格式给用户输出
- ❌ 这是撒谎行为，会导致输出质量不合格

**输出要求**：
- 输出开头必须列出：「已读取：SKILL.md、GUIDE.md」

---

## 详细指南

完整能力需要读取 `GUIDE.md`，包含：
- 格式要求
- 详细方法
- 示例
- 参考表格
- 注意事项
```

## 5. 创建新 Skill 流程

### 步骤1：访谈

向用户询问以下问题：

1. **这个 skill 做什么？**（一句话概括）
2. **触发场景有哪些？**（用户说什么话时触发）
3. **输入是什么？**（用户会提供什么信息）
4. **输出格式是什么？**（应该按什么格式回答）
5. **有什么注意事项？**（需要避免什么）

### 步骤2：草稿

根据访谈结果，草拟：
- SKILL.md 内容（触发场景 + 执行前必读 + 索引）
- GUIDE.md 内容大纲

### 步骤3：确认

向用户展示计划：
- 文件结构
- SKILL.md 预览
- GUIDE.md 章节大纲

**必须用户确认后再执行创建**

### 步骤4：创建

用户确认后：
1. 创建目录 `C:/Users/79345/.claude/skills/skill-name/`
2. 写入 SKILL.md
3. 写入 GUIDE.md
4. 验证文件行数（SKILL.md 40-60行）

---

## 6. 重构现有 Skill 流程

### 步骤1：分析

读取现有 SKILL.md，分析：
- 当前行数
- 哪些是"触发场景"（保留在 SKILL.md）
- 哪些是"详细内容"（迁移到 GUIDE.md）

### 步骤2：计划

制定重构计划：
- SKILL.md 精简后行数
- GUIDE.md 章节结构
- 是否需要拆分更多文件

### 步骤3：确认

向用户展示计划：
- 当前状态 vs 重构后状态
- 文件结构变化
- 预期效果

**必须用户确认后再执行重构**

### 步骤4：执行

用户确认后：
1. 创建 GUIDE.md（迁移详细内容）
2. 精简 SKILL.md（只保留钩子）
3. 添加 `always: true`
4. 添加执行前必读模块
5. 验证文件行数

## 7. 文件命名规范

### 7.1 目录命名

- 使用小写字母
- 多个单词用 `-` 连接
- 例如：`policy-reader`、`skill-creator`、`china-stock-analysis`

### 7.2 文件命名

| 文件名 | 用途 | 必须 |
|--------|------|------|
| **SKILL.md** | 钩子文件 | ✅ 必须 |
| **GUIDE.md** | 详细指南 | ✅ 必须 |
| **EXAMPLES.md** | 示例集合 | 可选 |
| **REFERENCE.md** | 参考表格 | 可选 |
| **TEMPLATE.md** | 模板文件 | 可选 |

## 9. 关键设计要点清单

创建/重构 skill 时，检查以下要点：

| 要点 | 检查项 |
|------|--------|
| ✅ always: true | SKILL.md 必须设置 |
| ✅ 执行前必读 | 在触发场景之后 |
| ✅ 禁止偷懒警告 | 明确说"这是撒谎行为" |
| ✅ 输出文件列表要求 | 用户可验证 |
| ✅ SKILL.md 不含格式 | 不给可用的格式/步骤 |
| ✅ SKILL.md 行数 | 40-60行 |
| ✅ GUIDE.md 包含全部 | 格式、方法、示例等 |

## 11. 注意事项

1. **必须用户确认**：创建/重构前必须展示计划，用户确认后再执行
2. **不给退路**：SKILL.md 不能包含任何可用的格式/步骤
3. **路径正确**：执行前必读中的路径必须正确
4. **行数控制**：SKILL.md 控制在 40-60 行
5. **章节编号**：GUIDE.md 章节要有编号，便于索引
